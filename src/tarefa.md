DESAFIO DE CONHECIMENTO  
DESENVOLVEDOR DE SISTEMAS – PLENO 
1. CENÁRIO 
Um cliente solicitou para o IATec a construção de um software de gerenciamento de 
agenda. No item 2, você encontra os requisitos que foram coletados com o cliente, que 
são necessários para o desenvolvimento do sistema. 
2. REQUISITOS 
1. O usuário deverá fazer login; 
2. Cada usuário terá sua própria agenda; 
3. A agenda terá seus eventos: 
4. Para criar um novo evento será necessário armazenar os seguintes dados: Nome, 
descrição, data, local e participantes; 
5. O evento poderá ser de um dos tipos: Exclusivo ou compartilhado. O evento 
compartilhado pode ser compartilhado com outros usuários e deverão aparecer em 
suas agendas. O evento exclusivo fica restrito apenas ao usuário que criou; 
6. Cada evento pode ser editado, podendo editar os participantes e editar o 
compartilhamento ou até mesmo remover ambos; 
7. Cada evento pode ser removido e deverá remover tudo que estiver atrelado a ele; 
8. O evento pode estar ativo ou não; 
9. Construção do DashBoard: 
10. Ao realizar o login no sistema, o usuário deverá ser redirecionado para a página de 
Dashboard 
11. A página deverá conter os seguintes filtros: data, hora e deverá poder filtrar por 
qualquer texto que o evento tiver; 
12. Deverá também conter três botões filtrando por eventos do dia, da semana, do mês; 
3. REQUISITOS TÉCNICOS OBRIGATÓRIOS 
O desenvolvimento deverá obedecer às seguintes regras: 
1. O Front-end da aplicação web deverá ser desenvolvido em Angular 
2. Será avaliado como foi feita a organização dos arquivos e organização do código 
3. O projeto deverá ter seus environments, modules e components devidamente 
criados; 
4. O Back-end deverá ser desenvolvimento em C# no framework .NET 10 
5. A api deverá ser REST ou REST FULL; 
6. A api deverá ser versionada; 
7. O desenvolvimento da api deve ser feito com Command Query Responsability 
Segregation; 
8. O banco de dados a ser utilizado pode ser de sua escolha, porém o gerenciamento 
deverá ser feito com Entity Framework Core; 
4. AVALIAÇÃO 
A avaliação será feita verificando se os requisitos do sistema foram contemplados 
juntamente aos requisitos técnicos. São desejáveis, por tanto não obrigatórios, os seguintes 
requisitos técnicos: Obedecer aos princípios do SOLID, Repositórios, injeção de dependência, 
criação de Data Transfer Object (DTOs) e AutoMapper. 
Fique livre para implementar melhorias que fortaleçam a experiência do usuário ou 
que enriqueçam a aplicação. Aproveite para mostrar seu conhecimento em áreas que não 
foram citadas explicitamente. 
A aplicação desenvolvida deverá ser apresentada ao entrevistador através de uma 
videoconferência na data acordada. Nessa data o código será avaliado e serão feitas perguntas 
com relação ao mesmo. Plágio do código desclassifica o candidato. 
Esta avaliação tem como foco principal entender qual é o nível de conhecimento do candidato
